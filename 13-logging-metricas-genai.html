1. Contexto e Propósito

Em sistemas tradicionais, logs são apenas texto: "User logged in". Em GenAI, logs são literalmente dinheiro. Cada token de entrada e saída tem um custo direto e marginal. Uma feature mal otimizada que entra em loop pode queimar centenas de dólares em minutos.

Segundo o relatório State of GenAI Observability 2024, 78% das empresas que não monitoram granularmente o uso de tokens têm surpresas negativas na fatura da nuvem. Além do custo, a latência em IA não é binária. O Time to First Token (TTFT) — o tempo até o usuário ver a primeira letra — é mais crítico para a percepção de velocidade do que o tempo total da resposta.

Tipo de Log Impacto no Negócio Métrica Essencial
Tradicional Auditoria de acesso user_id, status_code
GenAI Custo Direto & UX prompt_tokens, ttft, cost_usd

4. Mão na Massa: Instrumentação

Wrapper Python para logar métricas automaticamente no OpenTelemetry e LangFuse.

import time
import langfuse

# Setup do OTel
tracer = trace.get_tracer(__name__)
meter = metrics.get_meter(__name__)
token_counter = meter.create_counter("llm_tokens_total")

def call_llm_monitored(prompt, model="gpt-4", feature="summary"):
    with tracer.start_as_current_span("llm_call") as span:
        start_time = time.time()

        response = openai.ChatCompletion.create(...) 
        
        duration = time.time() - start_time
        usage = response['usage']

        # 1. Métricas para Prometheus/Grafana (Agregados)
        token_counter.add(usage['total_tokens'], {
            "model": model, 
            "feature": feature
        })

        # 2. Atributos no Trace para Debug (Individual)
        span.set_attribute("llm.model", model)
        span.set_attribute("llm.cost", calculate_cost(usage, model))

        # 3. Log rico para LangFuse (Observabilidade GenAI)
        langfuse.log_event({
            "name": "llm-generation",
            "input": prompt,
            "output": response.choices[0].message.content,
            "usage": usage,
            "metadata": {"feature": feature}
        })
        
        return response

Alertas de Anomalia

  • Spike de Custo: Gasto > $50 em 1 hora. (Pega loop infinito).
  • Latência Alta: P99 > 5s por 5 min. (Modelo engargalado).
  • Error Rate: > 5% de falhas. (Provider down).

Tagging de Features

Sempre logue feature="nome". Isso permite descobrir que o "Resumo de Menu" gasta 80% do budget, enquanto o "Chat Principal" gasta só 20%.

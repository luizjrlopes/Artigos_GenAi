3. Conceitos Fundamentais

Semantic Cache

Diferente de um cache exato (key-value), o cache semântico usa vetores. Ele entende que "Quero pizza" e "Tô com fome de pizza" são a mesma intenção (similaridade > 0.9) e retorna a resposta salva.

Hit Rate: 5% (Exato) -> 40% (Semântico)

Token Packing

Técnicas para reduzir o payload. Remover espaços, usar JSON minificado ou YAML, e remover stopwords. Pode reduzir custos em 20% sem perda de qualidade.

Model Cascade (Cascata)

Não use um canhão para matar uma mosca. Classifique a dificuldade da tarefa e roteie para o modelo mais barato capaz de resolvê-la.

4. Mão na Massa: Implementação

1. Cache Semântico com Redis

Implementação Python usando Redis VSS (Vector Similarity Search). 

import redis
from sentence_transformers import SentenceTransformer

r = redis.Redis(host='localhost', port=6379)
model = SentenceTransformer('all-MiniLM-L6-v2')

def get_cached_response(user_query, threshold=0.9):
    # 1. Gera embedding da query atual
    query_vector = model.encode(user_query).tobytes()

    # 2. Busca vetorial no Redis (Vizinhos mais próximos - KNN)
    result = r.ft("idx:responses").search(
        Query("*=>[KNN 1 @vector $vec AS score]")
        .return_field("response")
        .return_field("score")
        .dialect(2),
        {"vec": query_vector}
    )

    if result.docs:
        doc = result.docs[0]
        similarity = 1 - float(doc.score) # Converter distância para similaridade
        
        if similarity > threshold:
            print("Cache Hit! Economia: $0.03")
            return doc.response

    return None

Risco: Cache Stale

O usuário pergunta "Onde está meu pedido?". Se você cachear isso, vai responder "Na cozinha" para sempre. Nunca cacheie dados transacionais, apenas FAQs e conhecimentos estáticos.

Boas Práticas

  • Auditoria mensal de custos por feature.
  • Limpeza de contexto (remover msgs antigas).
  • Teste A/B de modelos em produção.

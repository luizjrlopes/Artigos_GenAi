1. Contexto e Propósito

Você mudou o prompt de recomendação de pratos para ser "mais engraçado". O time adorou. Mas será que o usuário vai comprar mais? Ou ele vai achar irritante e fechar o app? Em produtos digitais, opinião não escala. Dados escalam.

Segundo o relatório State of GenAI Experimentation 2024, 74% dos experimentos A/B em IA falham por falta de rigor estatístico ou coleta inadequada de métricas. Testar IA é mais complexo que testar a cor de um botão, pois a resposta é dinâmica e o impacto varia por contexto.

Teste Tradicional

  • Variável: Estática (Cor, Texto)
  • Métrica: CTR / Conversão
  • Complexidade: Baixa

Teste GenAI

  • Variável: Dinâmica (Prompt, Modelo)
  • Métrica: Conversão + NPS + Latência
  • Complexidade: Alta (Viés, Custo)

3. Conceitos Fundamentais

Sticky Sessions

Garante que o usuário João sempre veja a variante B durante o teste. Se ele ver A de manhã e B à tarde, os dados ficam sujos. Geralmente feito via hashing determinístico (`hash(user_id) % 100`).

Guardrail Metrics

Métricas que não podem piorar. Exemplo: Se a variante B aumenta conversão em 10%, mas aumenta a latência em 5s ou quebra o layout, o teste deve ser abortado.

Teste A/A: Validando o Pipeline

Antes de testar A vs B, teste A vs A (duas variantes iguais). Se houver diferença significativa nos resultados, seu sistema de randomização está quebrado.

4. Mão na Massa: Implementação

1. Router com Hashing Determinístico (Python)

Como dividir usuários consistentemente sem precisar de banco de dados. 

import hashlib
from enum import Enum

class Variant(Enum):
    CONTROL = "gpt-3.5-turbo" # Prompt atual
    TREATMENT = "gpt-4-turbo" # Novo prompt

def get_user_variant(user_id, experiment_salt="exp_desc_v1"):
    # Hash determinístico: O mesmo user sempre cai no mesmo bucket
    hash_input = f"{user_id}:{experiment_salt}"
    hash_val = int(hashlib.sha256(hash_input.encode()).hexdigest(), 16)

    # Split 50/50
    if hash_val % 100 < 50:
        return Variant.CONTROL
    return Variant.TREATMENT

# Logging é CRUCIAL para análise posterior
logger.info("exposure", extra={
    "user_id": user_id,
    "variant": variant.value,
    "timestamp": now()
})

2. Análise SQL (Pós-Experimento) 

SELECT
    variant,
    COUNT(DISTINCT user_id) as users,
    AVG(conversion_rate) as cvr,
    AVG(latency_ms) as avg_latency,
    -- Métrica de Guarda (NPS)
    AVG(nps_score) as avg_nps
FROM
    experiment_logs
WHERE
    experiment_id = 'exp_desc_v1'
GROUP BY
    variant;

6. Evidence & Ferramentas

Riscos Comuns

  • Efeito Novidade: O usuário interage mais só porque mudou. Espere 2 semanas para decidir.
  • Interferência: Rodar teste de Prompt junto com teste de UI. Impossível saber a causa da melhora.

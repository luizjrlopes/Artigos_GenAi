1. Contexto e Propósito

No mundo tradicional de microserviços, estamos acostumados com requisições rápidas: o app chama o backend, o backend consulta o banco e devolve o JSON em 50ms. Com GenAI, esse paradigma quebra.

Uma chamada para o GPT-4 pode levar 10, 30, até 60 segundos. Se você mantiver uma conexão HTTP síncrona aberta por tanto tempo, vai estourar timeouts em load balancers, travar a thread do cliente e frustrar o usuário que fica olhando para uma tela branca.

Propósito

Explorar padrões de design de API robustos para lidar com a latência inerente e a imprevisibilidade dos LLMs em produtos de escala.

2. Abordagem

O design de APIs GenAI exige repensar o ciclo de vida da requisição e resposta. Abaixo, os três padrões essenciais:

1. Streaming (SSE)

Visualização da resposta em tempo real, reduzindo a sensação de espera.

  • Reduz ansiedade
  • Permite cancelamento
  • Melhora TTFT

2. Async + Polling

Ideal para tarefas longas como relatórios e análises extensas.

  • Backend retorna 202 Accepted
  • Processamento em fila (SQS/Rabbit)
  • Polling ou Webhook

3. Circuit Breakers

Essenciais para resiliência quando o LLM falha.

  • Fallback para cache
  • Notificação clara
  • Registro de erro

3. Conceitos Fundamentais

Server-Sent Events (SSE)

Padrão em que o servidor mantém a conexão aberta e envia “chunks” de texto conforme são gerados. Simula o efeito “máquina de escrever”. 

from fastapi.responses import StreamingResponse
def generate_stream():
    for token in llm.stream():
        yield token

Time-to-First-Token (TTFT)

Métrica que mede o tempo até o usuário ver o primeiro caractere. Mais importante que a latência total!

  • TTFT: Impacto Alto
  • Latência Total: Impacto Médio
  • Throughput: Impacto Alto

Backpressure

Capacidade de absorver picos sem sobrecarregar o modelo via rate limiting. 

from slowapi import Limiter
limiter = Limiter(key_func=get_user_id)
@app.post("/chat")
@limiter.limit("10/minute")

4. Mão na Massa: Exemplo Prático

Passo a Passo

  1. Defina o padrão ideal (Streaming, Async, Sync).
  2. Implemente o endpoint (FastAPI, Flask).
  3. Adicione rate limiting e circuit breaker.
  4. Configure logs (TTFT, latência).
  5. Teste com cenários reais.

Checklist de Implementação

  • Endpoint streaming implementado
  • Endpoint async com polling/webhook
  • Circuit breaker configurado
  • Métricas de TTFT logadas
  • Testes de carga executados

Exemplo de Automação de Testes 

import requests
import time

def test_streaming():
    start = time.time()
    response = requests.post("http://localhost:8000/chat/stream", json={"message": "Oi"}, stream=True)
    first_chunk_time = None
    for i, chunk in enumerate(response.iter_lines()):
        if i == 0:
            first_chunk_time = time.time() - start
        print(chunk)
    print("TTFT:", first_chunk_time)

test_streaming()

5. Métricas, Riscos e Boas Práticas

Risco Impacto Mitigação
Timeout de Infra Quebra de conexão Keep-alive, SSE
Custo de Conexão Memória excessiva Limitar conexões
Rate Limit Excedido Erro 429 Rate limiting granular
Falha de LLM Queda de serviço Circuit breaker, Cache

Boas Práticas e Logs

  • Rate Limiting Granular: Por usuário, não por IP.
  • Cache Semântico: Verifique se a pergunta já foi respondida.
  • Logs Estruturados: Registre tudo.
  • Automação: Scripts para medir TTFT.

Sem Streaming (Síncrono) 

@app.post("/chat-sync")
def sync_chat(message: str):
    response = llm.generate(message) # 15s
    return {"response": response}

Métrica: 15s de tela branca → Usuário pensa que travou.

7. Reflexões & Próximos Passos

"Percebi que a maioria das falhas de GenAI em produção não é porque o modelo é ruim, é porque a API em torno dele é pior. Um usuário esperando 60 segundos com tela branca é a mesma coisa que um modelo que desistiu."

A Lição

Desenhar APIs para IA me fez reaprender HTTP. Coisas como chunks, buffers e event-streams viraram fundamentais.

Mesmo com streaming perfeito, coisas dão errado (alucinação, queda da OpenAI). Design robusto precisa de fallbacks.

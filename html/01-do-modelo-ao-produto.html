Do modelo ao produto

Como uma API de LLM vira funcionalidade em um app de entrega.

De um protótipo rápido em Jupyter para uma feature robusta lidando com latência e escala.

1. Contexto e Propósito

Nos últimos anos, integrar LLMs virou algo trivial: gere uma chave de API, escreva poucas linhas de código e voilà. Mas em um app de delivery processando milhares de pedidos por minuto, isso é só o começo.

O grande desafio? Transformar essa capacidade bruta em valor real para o usuário, sem quebrar o app com timeouts ou custos exorbitantes.

3. Conceitos Fundamentais

Termo Definição Contexto de Delivery
O Modelo (LLM) O "cérebro" (GPT-4, Llama 3). Stateless. A engine que processa o texto.
A Feature O pacote completo de software. Busca no DB + Prompts + API + UI.
Context Window Limite de tokens processados de uma vez. Foque nas últimas 20 avaliações.
Mudança de Mindset: Pare de tratar o LLM como oráculo. Veja-o como um componente transformador de texto em um pipeline clássico.

Código: Endpoint FastAPI

import os
from typing import List
from fastapi import FastAPI, HTTPException
from openai import OpenAI

app = FastAPI()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# Simulação de busca no banco
def get_recent_reviews(restaurant_id: str) -> List[str]:
    return [
        "A comida chegou fria, mas estava saborosa.",
        "Entrega super rápida! O sushi estava fresco."
    ]

def generate_summary_prompt(reviews: List[str]) -> str:
    reviews_text = "\n".join([f"- {r}" for r in reviews])
    return f"""
    Você é um assistente de delivery. Resuma em 3 linhas:
    Avaliações: {reviews_text}
    """

@app.get("/restaurants/{restaurant_id}/reviews/summary")
async def get_review_summary(restaurant_id: str):
    try:
        reviews = get_recent_reviews(restaurant_id)
        if not reviews:
            return {"summary": "Sem avaliações suficientes."}

        prompt = generate_summary_prompt(reviews)

        # Chamada ao LLM
        response = client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=[{"role": "user", "content": prompt}],
            max_tokens=150,
            temperature=0.5
        )

        return {
            "restaurant_id": restaurant_id,
            "summary": response.choices[0].message.content.strip()
        }

    except Exception as e:
        raise HTTPException(status_code=503, detail="Indisponível")

5. Métricas, Riscos e Boas Práticas

Latência

Chamadas de 2-10s geram abandono.

Solução: Cache (Redis) 24h.
Custos

Milhares de chamadas explodem a conta.

Solução: Rate Limiting & GPT-3.5
Confiabilidade

API cai ou alucina.

Solução: Circuit Breaker

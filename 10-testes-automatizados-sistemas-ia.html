1. Contexto e Propósito

"Como eu sei que a mudança no prompt não quebrou tudo?" Essa é a pergunta de 1 milhão de dólares. Em software tradicional, temos a segurança binária: assert result == expected. Em IA, o resultado muda a cada execução devido à natureza estocástica dos modelos, e "quase igual" pode ser aceitável ou catastrófico dependendo do contexto.

Segundo o relatório State of AI Testing 2024, 68% dos bugs críticos em produção só foram detectados por testes semânticos automatizados. Empresas que implementaram pipelines de testes para IA reduziram incidentes em 75%. Sem testes, você tem medo de mexer no prompt. Se tem medo de mexer, seu produto estagna e perde competitividade.

2. Abordagem: A Pirâmide de Testes IA

Não basta verificar se o JSON é válido. Precisamos de camadas de defesa que avaliem desde a estrutura até a semântica e a segurança (alucinações).

1. Commit / PR
2. Determinísticos
Schema, JSON válido, Estrutura
3. Semânticos (LLM-as-Judge)
Qualidade, Tom, Polidez
4. Alucinação & Robustez
Groundedness (RAG), Inputs Adversariais
5. Deploy Seguro

3. Conceitos Fundamentais

LLM-as-a-Judge

Usar um modelo mais forte (ex: GPT-4) para dar nota (0-5) para a resposta de um modelo mais rápido (ex: GPT-3.5). É caro, mas escala melhor que humanos e remove viés de equipe.

Golden Dataset

Um conjunto curado de "Perguntas + Respostas Ideais" (Gabarito). Serve para testes de regressão: "O novo prompt piorou as respostas antigas?"

Estratégia Vantagem Limitação
Igualdade Exata Fácil de automatizar Frágil para IA (quebra fácil)
Fuzzy Assert Tolerante a variações Pode perder sutilezas
LLM-as-Judge Avaliação real Custo e latência altos

4. Mão na Massa: Implementação com Pytest

Exemplo prático de um arquivo de teste que cobre estrutura, semântica e alucinação. 

import pytest

import json
from my_app import bot, gpt4

# 1. Teste Determinístico (Estrutura)
def test_recommendation_structure():
response = bot.ask("Estou com fome de pizza")
data = json.loads(response)

    # Garante que o contrato JSON foi respeitado
    assert "dishes" in data
    assert len(data["dishes"]) > 0
    assert "price" in data["dishes"][0]

# 2. Teste Semântico (LLM-as-a-Judge)
def evaluate_politeness(bot_response):
judge_prompt = f"""
Avalie se a resposta é educada.
Resposta: "{bot_response}"
Responda APENAS com 'SIM' ou 'NAO'.
"""
verdict = gpt4.generate(judge_prompt)
return verdict.strip() == "SIM"

def test_bot_is_polite():
# Teste de Robustez: Input agressivo
response = bot.ask("Essa comida é horrível!")
assert evaluate_politeness(response) is True

5. Métricas, Riscos e Mitigação

Risco: Custo do Teste

Rodar GPT-4 para avaliar 1000 prompts a cada commit vai falir o projeto.

Mitigação: Testes semânticos em Nightly Builds ou apenas em modelos menores.

Risco: Viés do Juiz

O LLM avaliador pode preferir respostas verbosas ou ter vieses próprios.

Mitigação: Prompt fixo para avaliação e múltiplos juízes.
